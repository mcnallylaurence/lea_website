Header rules & behaviour

  • Sticky on scroll: Header stays fixed at the top as the user scrolls down any page. The "Book a Free Call →" button must always be visible.
  • Background: Transparent on the homepage hero (so the full-screen image shows through), then shifts to white/opaque once the user scrolls past the hero. On all other pages: always white/opaque.
  • Active state: The current page's nav item should have an underline or slightly heavier weight so users know where they are.
  • What to remove vs current: Remove "Our Story" from nav (it becomes a homepage section). Remove "Testimonials" from overflow (renamed "Real Weddings" in primary nav). Remove "FAQs" from nav (move to footer). Remove "Elopement in Ireland" from nav (it's an SEO page — footer link only).

Stop the scroll. Establish Ireland + romance + credibility within 3 seconds. One clear primary action.

Layout

Copy

Squarespace implementation note

Squarespace does not natively support multi-filter gallery filtering. Options: (1) Use Squarespace tags + the Summary Block filtered by category — limited to one filter at a time. (2) Embed a simple custom JS filter using data attributes on the cards — works well and doesn't require leaving Squarespace. (3) Build the venue directory as a separate page in Airtable or Notion embedded via iframe — more powerful but adds complexity. Recommendation for Phase 1: single-filter by Region using Squarespace categories. Add style + guest count filters in Phase 2.

Action: Tag all venues in Squarespace before launch. Every venue page needs: Region tag, Style tag, Guest count range tag. This is a content task. Assign one person to go through all existing venue pages and tag them consistently.

Section 3 — Venue card grid

Layout

3-column grid on desktop, 2-column on tablet, 1-column on mobile. Each card is square or 4:3 ratio photo, with venue info below.

Each venue card contains

  • Full-width venue photo (hero image of the venue)
  • Venue name (bold)
  • Region badge (e.g. "Wild Atlantic Way")
  • Style tag (e.g. "Castle")
  • One-line description (max 10 words — teaser only)
  • Link: entire card is clickable → /venues/p/[slug]

Featured venues — show first, unpinned

The 6 venues from the homepage gallery (Cliffs of Moher, Dunluce, Killarney, Muckross, Rahinnane, Dunseverick) should appear first in the grid before the remaining venues, regardless of alphabetical or other sorting.

Show all / load more

Show 12 venues initially. "Load more" button reveals additional venues (or show all — depends on Squarespace capability). Do not paginate — couples should be able to browse without clicking through pages.

Individual venue page spec — /venues/p/[slug]

Purpose

This is where a couple decides "yes, this is the place." It needs enough information to create an emotional connection AND answer the practical questions before they book a call.

Every individual venue page must contain

  1. Photo gallery: Minimum 3 photos — the venue itself, a ceremony setup if available, and a couple portrait at the venue. Full-width hero image at top.
  2. Venue name & region
  3. Style and guest count tags
  4. Description: 2–3 paragraphs. What makes this venue special. The atmosphere. What a ceremony here looks and feels like. Written in second person ("you'll stand…").
  5. Practical info block:
    • Location / nearest town
    • Guest capacity (min / max)
    • Ceremony style (legal / symbolic / both)
    • Indoor / outdoor / both
    • Permit required? (yes / no)
    • Best season
  6. Which packages work here: List which of the 5 tiers are compatible with this venue. Link each tier name to /packages#[tier].
  7. Real couple story at this venue (if available): 1 featured photo + couple name + short quote. Links to /real-love.
  8. CTA block:
    • "Love this venue? Let's check availability." → /get-in-touch?venue=[venue-name]
    • Secondary: "← Back to all venues" → /venues
  9. Related venues: "You might also like…" — 3 venue cards from the same region. Each links to its own venue page.
Action: Audit all existing venue pages. Check each /venues/p/[slug] page currently on the site. For each one: does it have 3+ photos? A description? Practical info? If not, add this content before the new venues page goes live. A venue card that links to an empty or stub page loses trust.

## PART 1: BRAND IDENTITY

### Voice
You sound like a trusted friend who has actually done this. You are:
- **Authentic:** Written from lived experience (Yasmeen and Laurence's story is real)
- **Warm & encouraging:** Acknowledge the couple's emotions and concerns
- **Clear & practical:** Explain what things cost, how long they take, what's needed
- **Slightly playful, never cheesy:** Joyful but subtle, not forced
- **Honest about differences:** Don't hide complexity; lean into what makes Ladybird different

### The Two Core Stories You Always Lead With (When Relevant)
1. **Founder story:** Yasmeen and Laurence met at a bar called Ladybird in Atlanta. They planned their own Irish wedding from the US. They built Ladybird Ever After because every couple deserves someone who has actually done this.
2. **USD contract story:** Ladybird is the ONLY US-based operator with a USD contract. One invoice. One American company. Zero international payment headaches. You never send money to Ireland.

### Tone by Context
- **Blog posts & social:** Conversational, friendly, personal pronouns, short sentences, contractions
- **Pricing & packages:** Warm but structured, bulleted lists, explaining the "why"

### What You DON'T Sound Like

## PART 2: STYLE RULES

### Content Length Standards
- **Blog posts (standard):** 1,200–2,000 words
- **Pillar posts:** 2,000–3,000 words
- **Quick tips/short posts:** 600–900 words
- **Email:** 150–250 words per email
- **Page copy (hero):** 50–80 words
- **Section intros:** 80–150 words

### Heading Hierarchy (Critical)
- **H1:** One per page. Playfair Display 300. No italics. Page title only.
- **H2:** Playfair Display 300. Section headings. CAN include `<em>` for italics in gold color.
- **H3:** Playfair Display 300. Subsections. Break up long sections.
- **Body:** Alice 300 font. 0.95–1rem. 1.5 line-height.

### Heading Style Rules
- Capitalize normally (Title Case). Never all caps.
- Use italics sparingly in H2s only: `<em>Italicized Words</em>` (colored gold)
- Example: "Irish Ceremony *Traditions* for American Couples"

### Paragraph & Sentence Rules
- **Paragraph length:** 2–4 sentences max. Shorter is better.
- **Sentence length:** 10–20 words average. Vary between short and medium.
- **Voice:** Use contractions ("you're," "we'll," "don't")

### Lists (Format Rules)
- **Bulleted:** Use for highlights, features, quick tips. Preceded by intro line.
- **Numbered:** Use only when order matters (steps, processes).
- **Max items:** 5–7 per list
- **Capitalization:** Title Case first word of each bullet
- **Punctuation:** Period if complete sentence, no period if fragment

#### Example:
✓ "Here's what's included in every package:
• US-based contract in USD
• Unlimited planning calls
• Legal marriage guidance
• Venue curation and booking
• Welcome dinner planning"

### Links & CTAs
- **Anchor text:** Descriptive, not "click here"
- **Examples:**
  - ✓ "Explore our [four core packages](/packages)."
  - ✓ "Read our complete [guide to getting married in Ireland](/blog/getting-married-in-ireland-as-an-american)."
- **Button copy:** Action-oriented, short (2–4 words)
  - ✓ "Let's plan this"
  - ✓ "See our packages"
  - ✗ "Click to continue"

### Images & Alt Text
- **Every image needs alt text.** 8–15 words, descriptive.
- Formula: "[What is shown] in [Location/Context] for [Purpose]"
- Examples:
  - ✓ "Bride and groom at Cliffs of Moher elopement ceremony"
  - ✓ "Stone archway at Dunluce Castle wedding venue in County Antrim"
  - ✗ "wedding photo"

### Typography Rules
- **Fonts:** Playfair Display (headings only) + Alice (body only)
- **Colors:** Use brand colors only
  - Green: #5B6239 (primary buttons, accents)
  - Gold: #B8964E (accent, icons, decorative lines)
  - Gold light: #D4AF71 (italic text in headings)
  - Cream: #F5F0E8 (section backgrounds)
- **Formatting:**
  - ❌ Never use em-dashes (—). Use en-dashes (–) or rephrase.
  - ❌ Never use "and/or." Choose one or rewrite.
  - ❌ Never use ALL CAPS for emphasis. Use bold or italics.
  - ❌ Never use special Irish characters (ó, é, etc.) in page/package names
  - ❌ No underline for non-linked text
  - ❌ Never chain multiple parenthetical clauses

### Common Words to Avoid
| Avoid | Use Instead |
|-------|-------------|
| "Leverage," "synergize," "streamline" | "Use," "bring together," "simplify" |
| "Curate," "artisan," "bespoke" (overused) | "Select," "handpicked," "custom" |
| "Journey," "adventure" (in wedding context) | "Plan," "celebrate," "experience" |
| "Magical," "enchanted," "fairytale" | "Beautiful," "meaningful," "real" |

### Standard Blog Post Structure
1. **Opening (50–100 words):** Acknowledge the question / pain point. Build curiosity.
   - Start with: "You're probably wondering..." or "Here's the thing..." or "Let's be honest..."
2. **Problem/Context (150–200 words):** Explain why this matters. Make it relatable.
3. **Solution/How-to (600–1,000 words):** Break into H2 sections. Use lists, examples, practical steps.
4. **Social proof/testimonials (100–150 words):** Real couple quote or stat from 198 deals
5. **CTA (50–80 words):** "Ready to plan?" Link to packages, get-in-touch, or relevant page

✓ "Here's the thing: getting married in Ireland is different from marrying at home."

### Blog Post Types & Approaches

#### Type 1: "How To" / Guide Posts
**Target:** Couples in awareness/consideration phase
**Structure:** Problem → Steps → Examples → CTA
**SEO focus:** "How to..." keywords
**Length:** 1,200–2,000 words

Example: "How to Get Married in Ireland as an American"
- H1: How to Get Married in Ireland as an American | 2026 Guide
- H2s: Legal Requirements, Visa & Residency, Ceremony Options, Timeline, Common Mistakes
- Each H2: 150–300 words, 1–2 example scenarios

#### Type 2: Comparison / "vs." Posts
**Target:** Couples comparing options or competitors
**Structure:** Option A explanation → Option B explanation → Comparison table → Recommendation → CTA
**SEO focus:** "X vs. Y" or "Should I..." keywords
**Length:** 1,200–1,800 words

Example: "Elopement vs. Wedding in Ireland: Which Is Right for You?"
- H1: Elopement vs. Wedding in Ireland: Which Is Right for You?
- H2s: What's an Elopement?, What's a Wedding (Small)?, Key Differences, Cost Comparison, Real Couple Stories
- Include: Comparison table, pros/cons, couple examples

#### Type 3: "Why" / Thought Leadership Posts
**Target:** Couples in the dreaming phase, differentiating Ladybird
**Structure:** Insight → Supporting details → Real examples → CTA
**SEO focus:** "Why choose," "benefits of," brand positioning
**Length:** 1,000–1,500 words

Example: "Why Marrying in Ireland (and Why You Should Hire a Planner)"
- H1: Why Marrying in Ireland | Complete Guide to Planning From Abroad
- H2s: The Draw of Ireland, International Complexity, Why Planners Matter, What We Handle, Getting Started
- Include: Founder story, 198 deals stat, USD contract story

#### Type 4: List / Listicle Posts
**Target:** Quick-reference searchers, social sharing
**Structure:** Intro → Numbered items (each with explanation + photo) → CTA
**SEO focus:** "Best," "top," "must-know" keywords
**Length:** 1,000–1,600 words

Example: "10 Most Stunning Castle Venues in Ireland for Weddings"
- H1: 10 Most Stunning Castle Venues in Ireland for Elopements & Weddings
- H2s: [Castle 1], [Castle 2], ... [Castle 10] (each with story, photo, link to venue page)
- Include: Cost estimates, guest capacity, why couples choose each

#### Type 5: Expert Q&A / FAQ Expansion
**Target:** Couples with specific questions
**Structure:** Question → Expert answer → Real example → Related reading
**SEO focus:** Long-tail question keywords
**Length:** 800–1,200 words

Example: "Should I Get Legally Married in Ireland or at Home?"
- H1: Should I Get Legally Married in Ireland or at Home?
- H2s: The Legal Path in Ireland, The US Legal Path, Advantages of Each, Real Stories, Our Recommendation
- Include: Founder advice, couple examples, practical checklist

## PART 10: VENUE CARDS

When a blog section mentions specific Ireland venues (cliffs, castles, parks, etc.), replace plain hyperlinks with **interactive venue cards**. Cards are pulled live from the Squarespace venue collection — they only work when the blog is live on the site, not in local file preview.

### How venue cards work

The site stores venues at `/venue?format=json&count=300`. Each venue item has a `fullUrl` like:
```
https://www.ladybirdeverafter.com/venue/p/cliffsofmoher
```
The last URL segment (e.g. `cliffsofmoher`) is the slug used to identify venues. **No hyphens, all lowercase.**

### HTML placeholder

Place this immediately after the paragraph that mentions the venues:

```html
<div class="venue-cards" data-venue-cards="slug1,slug2,slug3">
  <div class="venue-cards-loading">Loading venues&hellip;</div>
</div>
```

- List slugs in the same order you want the cards to appear
- 2-card rows → the grid auto-collapses to 2 columns
- 1 card → grid collapses to 1 column
- If a slug doesn't match, the card silently disappears

### Known venue slugs (confirmed URLs)

| Venue | Slug |
|-------|------|
| Cliffs of Moher | `cliffsofmoher` |
| Dunquin Pier | `dunquinpier` |
| Dunluce Castle | `dunlucecastle` |
| Rahinnane Castle | `rahinnanecastle` |
| Ross Castle | `rosscastle` |
| Killarney National Park | `killarneynationalpark` |
| Howth Cliff Walk | `howthcliffwalk` |
| JR Mahon's (Dublin pub) | `jrmahons` |

To find slugs for other venues: visit `https://www.ladybirdeverafter.com/venue/p/[name]` and check the last URL segment, or browse `/venue?format=json` in a browser when on the live site.

### JavaScript to include before `</body>`

Add this script block once at the bottom of every blog post that uses venue cards:

```html
<script>
(function () {
  function esc(str) {
    return String(str).replace(/&/g,'&amp;').replace(/</g,'&lt;').replace(/>/g,'&gt;').replace(/"/g,'&quot;');
  }

  function buildVenueCard(item) {
    const title  = item.title || 'Venue';
    const href   = item.fullUrl || '#';
    const imgSrc = item.assetUrl || item.thumbnailUrl || null;
    const county = (item.tags || []).find(t => t.match(/^(County|Co\.?)\s/i)) || '';

    const imgHtml = imgSrc
      ? `<img class="venue-card-img" src="${esc(imgSrc)}" alt="${esc(title)}" loading="lazy">`
      : `<div class="venue-card-img-ph"><svg viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="1.2"><rect x="3" y="3" width="18" height="18" rx="1"/><circle cx="8.5" cy="8.5" r="1.5"/><polyline points="21 15 16 10 5 21"/></svg></div>`;

    return `<a href="${esc(href)}" class="venue-card">
      ${imgHtml}
      <div class="venue-card-body">
        ${county ? `<div class="venue-card-county">${esc(county)}</div>` : ''}
        <div class="venue-card-title">${esc(title)}</div>
        <div class="venue-card-rule"></div>
        <span class="venue-card-cta">View venue</span>
      </div>
    </a>`;
  }

  async function loadVenueCards() {
    const grids = document.querySelectorAll('[data-venue-cards]');
    if (!grids.length) return;

    let allVenues = [];
    try {
      const res = await fetch('/venue?format=json&count=300');
      if (!res.ok) throw new Error();
      const data = await res.json();
      allVenues = data.items || data.products || [];
    } catch (e) {
      grids.forEach(g => g.innerHTML = '');
      return;
    }

    // Match on the last segment of fullUrl (e.g. "cliffsofmoher")
    const bySlug = {};
    allVenues.forEach(item => {
      const slug = (item.fullUrl || '').split('/').filter(Boolean).pop();
      if (slug) bySlug[slug.toLowerCase()] = item;
    });

    grids.forEach(grid => {
      const slugs = grid.dataset.venueCards.split(',').map(s => s.trim().toLowerCase());
      const matched = slugs.map(s => bySlug[s]).filter(Boolean);
      if (!matched.length) { grid.innerHTML = ''; return; }
      grid.innerHTML = matched.map(buildVenueCard).join('');
      if (matched.length === 2) grid.style.gridTemplateColumns = 'repeat(2, 1fr)';
    });
  }

  loadVenueCards();
}());
</script>
```

### When NOT to use venue cards

- City elopement sections where the "venue" is a general area (e.g. Dublin Georgian quarter) — use a plain link instead
- Multi-location day sections — these aren't single-venue pages
- Any venue that doesn't have a dedicated page on the site

### What to avoid

**Never** turn venue mentions into plain hyperlinks when a card is available. Cards drive engagement, reduce bounce rate, and link directly to venue pages that convert. Plain links are a fallback only when no venue page exists.

- **CTA block** → sends readers to browse packages (browsing intent)
- **HubSpot form** → captures leads ready to start a conversation (contact intent)
